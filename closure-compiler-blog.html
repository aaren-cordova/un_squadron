No you don't, but you should. You get better type checking at compile time, You should be documenting your code anyway. Is the closure library required to run the closure compiler? No. The compiler and the library we're written to work independently of one another. That being said, the closure library is written with the compiler in mind. That means more type annotations, more data type checks, and more compression.
